## Story

As a **maintainer re-running `sdlc init --adopt` (intentionally, or by mistake)**,
I want **re-runs to be no-ops on already-adopted state, and explicit conflict-resolution flows when a canonical target slot is already occupied** — namely: (1) a completed adopt re-run recognises existing symlinks from `adopted-symlinks.json`, Pass 2 asks ONLY about new candidates, Pass 3 stamps ONLY new adoptions, and the journal records an `adopt_re_run` summary; (2) Pass 2 hitting a target that is a REAL file prompts `[s]kip / [b]ackup-and-replace / [d]ifferent-target`; (3) Pass 2 hitting a target that is a DIFFERENT symlink prompts `[s]kip / [r]eplace / [d]ifferent-target` (recording BOTH a removal AND an accept event); (4) a partial adopt (Pass 1 done, Pass 2 interrupted) resumes Pass 2 from the first un-decided candidate and tells the user which were already decided —
so that **adopt-mode is safe to re-run and predictable when artifacts collide (FR2 idempotency, PRD §275 conflict cases), while the NFR-REL-6 source-untouched invariant continues to hold byte-for-byte across every re-run and conflict path.**

1. **Re-run is a no-op on already-adopted state + `adopt_re_run` summary (AC: epics.md:1907-1914).** Given a project where `sdlc init --adopt` completed once, when it is re-run: (i) Pass 1 re-detects the same artifacts (or its result is reused from `adopt-report.json` per D1) and existing symlinks are recognised via `adopted-symlinks.json`; (ii) Pass 2 skips every artifact whose canonical target is already in the manifest's `recorded_targets` and offers/auto-accepts ONLY new candidates — the interactive `confirm`/conflict callbacks are NOT invoked for already-adopted targets (B); (iii) Pass 3 stamps ONLY new adoptions (no duplicate `imported_from_existing` event or sidecar overwrite for an already-stamped artifact — absorbs CR3.4-W8 per D6); (iv) the journal records ONE `adopt_re_run` event with payload `{new_adoptions: N, skipped_existing: M}` (D3) via the event-only SYNC API. A re-run with zero new candidates ⇒ `new_adoptions=0`, no new `symlink_accepted`/`imported_from_existing` events, source bytes byte-identical.

2. **Conflict: target is a REAL file → `[s]kip / [b]ackup-and-replace / [d]ifferent-target` (AC: epics.md:1916-1924, PRD §275).** Given Pass 2 attempts to symlink to a canonical target that already exists as a REAL file (not a symlink), when the conflict surfaces, the user is prompted (verbatim): `Target <path> already exists as a real file. Options: [s]kip / [b]ackup-and-replace / [d]ifferent-target`. Then: `s` skips the candidate (no symlink, no record, optional skip-warn); `b` moves the existing real file to `.claude/state/adopt-conflicts/<timestamp>/<name>.bak` (byte-preserving rename, destination guarded by `assert_path_under_claude`, E/D5) and THEN creates the symlink + records the mapping + journals `symlink_accepted`; `d` re-prompts for a different canonical target path (validated under root via `is_target_under_root`, `_symlink.py:46`) and re-attempts the conflict check against the new slot. The conflict detection distinguishes real-file from symlink-conflict (C). In non-interactive mode (`confirm is None`) a real-file conflict skips-and-warns (no destructive backup without explicit consent).

3. **Conflict: target is a DIFFERENT symlink → `[s]kip / [r]eplace / [d]ifferent-target`, BOTH events journaled (AC: epics.md:1926-1929).** Given Pass 2 attempts to symlink to a canonical target that is already a symlink pointing at a DIFFERENT source (NOT `ALREADY_CORRECT`), when the conflict surfaces, the prompt is (verbatim): `Target <path> is already a symlink to <other-source>. Options: [s]kip / [r]eplace / [d]ifferent-target` (where `<other-source>` is the resolved current link target). Then: `s` skips; `r` removes the old symlink AND creates the new one, recording BOTH events in the journal — a removal event (D4, propose `symlink_replaced` payload `{target, old_source}`) AND the standard `symlink_accepted` for the new mapping, plus the manifest is updated (the old mapping for that target, if present, is replaced by the new one); `d` re-prompts for a different target. The `ALREADY_CORRECT` case (same source) is unchanged — idempotent record, never prompted (B).

4. **Partial-adopt resume: Pass 2 resumes from the first un-decided candidate + user informed (AC: epics.md:1931-1933, FR2; D1).** Given a partial adopt (Pass 1 completed → `passes_completed=(1,)` persisted; Pass 2 interrupted), when re-run: (i) Pass 1's result is honoured per D1 (reused from the report under the frozen skip-completed cursor — recommended — OR re-run if D1=(b)); (ii) Pass 2 resumes from the first un-decided candidate — already-recorded targets in `adopted-symlinks.json` are skipped (so candidates decided before the interruption are not re-offered, B), and offering continues from the first not-yet-recorded offerable artifact; (iii) the user is informed which candidates were already decided (e.g. a warn line per skipped-because-already-adopted candidate, or a summary count) via the injected `WarnCallback` (`adopt/` must not `print`). The frozen `_validate_resume_cursor` contiguous-prefix guard (`driver.py:111`) is preserved (or amended per D1 with a DAG note).
