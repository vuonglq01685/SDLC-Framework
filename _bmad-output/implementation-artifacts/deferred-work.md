- **EPIC-2A-DEBT-PANEL-V1-PROCESS-LOCAL-SEQ** (P1 of Story 2A.3 review) — Panel dispatch's `monotonic_seq` is allocated via a process-local async lock + cached counter in `dispatcher/_panel_helpers.py:_allocate_seq`. - **EPIC-2A-DEBT-JOURNAL-NULL-AFTER-HASH** (P19 partial of Story 2A.3 review) — `JournalEntry.after_hash` is a non-null sha256-pattern string (`src/sdlc/contracts/journal_entry.py:25`). - **EPIC-2A-DEBT-PANEL-ORPHAN-NOTE** (P4 informational of Story 2A.3 review) — `dispatch_panel` no longer uses `BoundedDispatcher.dispatch_many` for parallel members; it uses inline `asyncio.Semaphore(max_parallel_agents)` + `gather(return_exceptions=True)` to prevent orphan coroutines on first failure (the `dispatch_many` API uses `gather(return_exceptions=False)` which leaves siblings running per Python asyncio docs).
