## Deferred from: code review of 1-1-project-bootstrap-with-uv-init-hatchling (2026-05-07)

- **`license = { text = "TBD" }` is not a valid SPDX identifier** тАФ `pyproject.toml:6`. PyPI/twine increasingly reject non-SPDX license text. Spec Task 3.1 authorizes "TBD" as a placeholder for the substrate-only commit. **Owner:** later v0.2 chore (LICENSE file authoring).
- **sdist contains 1085 non-package files (3.3 MB for a 90-byte package)** тАФ hatchling sdist target has no exclude list. Wheel correctly contains only `sdlc/__init__.py`, but sdist sweeps `_bmad/`, `_bmad-output/`, `.claude/`, `docs/`, `tests/`, etc. Spec Task 4.4 explicitly defers full sdist suppression (`[tool.hatch.build.targets.sdist] exclude = ["**"]`) to Story 1.3 release.yml. Patch P2 (gitignoring scratch dirs) reduces blast radius in the interim because hatchling sdist defaults to "everything not gitignored." **Owner:** Story 1.3 (CI/CD pipelines + release.yml).
- **`__version__` duplicated between `pyproject.toml` and `src/sdlc/__init__.py`** тАФ both hardcoded as `"0.0.0"`. Future bumps will drift. Spec Task 3.3 explicitly chose static hardcoded over `importlib.metadata.version("sdlc-framework")` to keep AC2 simple. **Owner:** ADR-001 future revision (Story 1.5 authors the ADR; the runtime switch is a v1.x candidate).
- **AC1.4 literal wording vs intent** тАФ `.python-version` AC says "declares `>=3.10`" but actual value is the pin `3.12`. Dev Notes acknowledge "the file typically holds an exact pin like 3.10 or 3.12; the constraint is enforced by `requires-python`." Spec wording could be tightened: "`.python-version` pins a Python version satisfying `>=3.10`; floor is enforced by `requires-python`." **Owner:** spec author / next epic-1 retrospective.

## Deferred from: code review of 1-4-pre-commit-config-module-boundary-enforcement-hook (2026-05-08)

- **No CI enforcement of `pre-commit run --all-files`** тАФ `.github/workflows/`. Without CI hookup, devs without `pre-commit install` bypass the chain entirely. Already deferred per ADR-010 ┬з"Deferred". **Owner:** future CI hardening story (likely after Story 1.5 mkdocs lands).
- **Pre-commit `rev:` pin drift vs `uv.lock`** тАФ `.pre-commit-config.yaml:25`. The `astral-sh/ruff-pre-commit@v0.15.12` rev pin must be manually kept in sync with the ruff version locked by `uv.lock`. ADR-010 ┬зConsequences explicitly accepts this as a manual-review burden. **Owner:** future "rev sync" automation story OR a pre-commit hook that asserts the two match.
- **Hook ordering: hygiene hooks run after boundary-validator** тАФ `.pre-commit-config.yaml:68-82`. Universal-newlines reading neutralizes the LOC count concern today (CRLF and LF give identical counts), but the principle holds for any future content-affecting hook. **Owner:** revisit when adding a new hygiene hook that mutates content.
- **`tests/conftest.py` injects `scripts/` into `sys.path` globally** тАФ `tests/conftest.py`. Pollutes test-collection namespace. Cleaner: make `scripts/` an importable package or scope the `sys.path` injection to a fixture. **Owner:** when scripts/ grows beyond two files OR when Story 2A-2 lands `validate_specialists` proper.
- **`scripts/` directory not in MODULE_DEPS** тАФ `scripts/check_module_boundaries.py`. Scripts can import any sdlc.* module without enforcement. Acceptable for dev-tooling today; revisit if scripts grow into shared library code. **Owner:** the story that adds the third "production-leaning" script under `scripts/`.
- **`check-added-large-files --maxkb=500` may eventually fire on `uv.lock`** тАФ `.pre-commit-config.yaml:79-80`. Cumulative dep churn typically pushes lockfiles past 500KB over time. Fix path is "add `exclude: ^uv\.lock$` or bump `--maxkb=N`". **Owner:** addressable when it actually fires on a commit.
