## Deferred from: code review of 1-1-project-bootstrap-with-uv-init-hatchling (2026-05-07)

- **`license = { text = "TBD" }` is not a valid SPDX identifier** — `pyproject.toml:6`. PyPI/twine increasingly reject non-SPDX license text. Spec Task 3.1 authorizes "TBD" as a placeholder for the substrate-only commit. **Owner:** later v0.2 chore (LICENSE file authoring).
- **sdist contains 1085 non-package files (3.3 MB for a 90-byte package)** — hatchling sdist target has no exclude list. Wheel correctly contains only `sdlc/__init__.py`, but sdist sweeps `_bmad/`, `_bmad-output/`, `.claude/`, `docs/`, `tests/`, etc. Spec Task 4.4 explicitly defers full sdist suppression (`[tool.hatch.build.targets.sdist] exclude = ["**"]`) to Story 1.3 release.yml. Patch P2 (gitignoring scratch dirs) reduces blast radius in the interim because hatchling sdist defaults to "everything not gitignored." **Owner:** Story 1.3 (CI/CD pipelines + release.yml).
- **`__version__` duplicated between `pyproject.toml` and `src/sdlc/__init__.py`** — both hardcoded as `"0.0.0"`. Future bumps will drift. Spec Task 3.3 explicitly chose static hardcoded over `importlib.metadata.version("sdlc-framework")` to keep AC2 simple. **Owner:** ADR-001 future revision (Story 1.5 authors the ADR; the runtime switch is a v1.x candidate).
- **AC1.4 literal wording vs intent** — `.python-version` AC says "declares `>=3.10`" but actual value is the pin `3.12`. Dev Notes acknowledge "the file typically holds an exact pin like 3.10 or 3.12; the constraint is enforced by `requires-python`." Spec wording could be tightened: "`.python-version` pins a Python version satisfying `>=3.10`; floor is enforced by `requires-python`." **Owner:** spec author / next epic-1 retrospective.

## Deferred from: code review of 1-2-pyproject-toml-quality-gates-configuration (2026-05-07)

- **`filterwarnings = ["error"]` third-party escape hatch (YAGNI)** — `[tool.pytest.ini_options]` (`pyproject.toml:120`). No transitive `DeprecationWarning` trips today; the strict `error`-on-warning discipline is intentional NFR-MAINT-1 enforcement. Preemptive broad ignores would weaken the gate before evidence. **Owner:** when a real third-party `DeprecationWarning` first fails CI, add a precise targeted ignore (`default::DeprecationWarning:specific.module`) — not a regex blanket.
- **`required-imports = ["from __future__ import annotations"]` will fire on empty `__init__.py` and future `tests/conftest.py`** — `[tool.ruff.lint.isort]` (`pyproject.toml`) and the `tests/**` per-file-ignores list does not exempt `I002`. Architecture intent is that every Python file starts with the future-import; this is intentional discipline, not a bug. **Owner:** revisit if any sub-package's empty `__init__.py` becomes a real friction point.
- **ADR citations to `Architecture §N` and `PRD §N` lack hyperlinks** — Across `docs/decisions/ADR-002-*.md`, `ADR-003-*.md`, `ADR-004-*.md`. Reviewers cannot click through to verify claims. **Owner:** Story 1.5 (ADR-011 / mkdocs ADR template) — rewrites ADRs into canonical numbered template with cross-references.
- **Missing `py.typed` marker on `src/sdlc/`** — Without the marker, downstream consumers importing `sdlc` get `[import-untyped]` from mypy despite full annotations. Project is pre-release (`__version__ = "0.0.0"`); not a current AC. **Owner:** first publishable release (likely v0.1.0 or later) — `touch src/sdlc/py.typed` plus hatch wheel-include.
- **`xfail_strict = true` × `filterwarnings = ["error"]` interaction speculative** — An `@pytest.mark.xfail` test that emits any warning becomes a hard error before xfail logic evaluates. No concrete test triggers this today. **Owner:** address when first xfail-marked test emits a warning; per-test `@pytest.mark.filterwarnings` is the standard escape.
