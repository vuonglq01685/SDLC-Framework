## 3. What Did Not Go Well — Top 6 Patterns

### Pattern 1: Tautological / Placebo Tests (Critical)

Tests passing for the wrong reason — mocks subverting the very behavior the test claims to exercise. Identified in Stories **1.13, 1.14, 1.16, 1.19, 1.20** (nearly half the epic).

- 1.13: "yields control" test asserts `await something_that_was_never_blocked()` — tautological
- 1.14: "ignores non-engine" test creates a mock with no engine module → test always passes
- 1.16: "path-validation rejects traversal" monkeypatches *past* the path validator → test passes when validator is disabled
- 1.19: redundant assertion duplicating an immediately-prior assert
- 1.20: placebo "path-validation test" not actually testing path traversal

Root cause analysis: writing tests *after* implementation produces tests that match the code that ran, rather than spec'ing the behavior. Coverage of 93.37% is therefore over-stated relative to behavioral coverage.

### Pattern 2: POSIX-Only Modules + Windows mypy/coverage Friction

Stories **1.9, 1.10, 1.11, 1.12, 1.14, 1.16** — each POSIX-only module re-discovered the same pattern: `mypy.overrides warn_unreachable=false`, `coverage.omit`, `if sys.platform != "win32":` partial-branches. Stabilized at 1.10 but not abstracted — 5 stories paste the same pattern. Cost per POSIX module: ~1 ADR override + 1 mypy override + 1 coverage omit.

### Pattern 3: LOC-Cap Fights and Boundary-Validator Self-Grazes

Stories **1.4, 1.11, 1.15, 1.16, 1.19** — test files repeatedly grazed the 400-line cap. Story 1.11 `writer.py` landed at 236 LOC vs. the 200-LOC spec budget (residual deviation). `check_module_boundaries.py` of Story 1.4 itself failed its own LOC cap, requiring docstring trimming to pass. Architectural smell: gate-tools may need exception lists or higher LOC budgets.

### Pattern 4: Pydantic Lax-Mode Coercion Bugs

Stories **1.7, 1.8, 1.13** — `Literal[1]` accepted `True` and `1.0`; integer fields accepted `bool` and `float`; `4.5 → 4` silent float coercion. Required `Field(strict=True)` per field. This should be a project-wide default rather than per-field discipline.

### Pattern 5: Review-Patch Volume Increased Toward End of Epic

Patches per story: 1.5 = 15 (cosmetic) → 1.20 = **33 (real-bug)** → 1.18 split into two stories due to 1717 lines. 1.18 was an outlier in test-design difficulty (CLI follow-mode, time budgets, KI injection); 1.20 was natural — recovery code path is edge-case-rich. Pre-review tooling could reduce per-story burden.

### Pattern 6: Linter AST Blind Spots

Linters consistently missed: alias imports (`import os as o`), `if TYPE_CHECKING:` skip, `from sdlc import engine` form, `mode=` kwarg in `open()`, `seek()+write()` nested in lambdas, `os.open` / `Path.open` / `os.truncate` bypass paths. Each linter expanded coverage *after* a reviewer flagged the gap. A shared linter test harness (`TestVisitorDirect` base) could reduce duplicate work.
