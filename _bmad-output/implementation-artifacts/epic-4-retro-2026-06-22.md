## 3. Challenges & Recurring Themes

1. **Defer backlog (Project Lead's top concern) — ~30 open CR4.x-Wn.** Shape is layered, not 30 independent problems:
   - **Theme B — `EPIC-4-DEBT-AUTO-REAL-DISPATCH` (root):** production `sdlc auto` hard-aborts on the real runtime; the whole STOP fan-out + brainstorm + mad-mode are proven under `SDLC_USE_MOCK_RUNTIME=1` only. **≥7 defers gated behind it** (CR4.6-W2, CR4.7-W4/W6/W7, CR4.10-W1, CR4.11-W1, AUTO-BRAINSTORM-REAL-SIGNAL). Most-severe effect: **CR4.7-W6** — STOP trigger 6 (high-risk interception) does **not** protect the production loop yet, despite green tests, because the prod dispatch path never threads `auto_loop_mode=True`.
   - **Theme C — security (independent):** CR4.7-W7 token replay (Critical), CR4.12-W1 symlink/path-containment (HIGH×2), CR4.7-W1 secret_exfil regex/ReDoS.
   - **Theme D — registry coverage gaps:** CR4.8-W2 cross-trigger precedence untested; CR4.2-W3 non-sticky halt across runs (5.19 dashboard would lose the halt banner).
2. **Spec-prose vs frozen-code naming drift (escalating named trap class).** epics.md said "synthesizer" but code had `requirement-synthesizer` (4.10-C1); said `mad_resolution` but code had `auto_mad_resolve` (4.11-C1); the DAG said "reuse the replan invalidation seam" but `invalidate_record` couldn't satisfy the AC (4.12-D1). ADR-028 §3 tracks the real shipped names; DAG prose still carries stale names. Standing rule: **verify every load-bearing symbol against source; honor code spelling, not prose.**
3. **win32 "asserted-not-measured" gap.** Windows dev host can't import POSIX-only `io_primitives` at conftest → full pytest / coverage / freeze are asserted locally and first truly execute on CI. Recurred as CR4.6-W1 / CR4.8-W1 (resolved) / CR4.9-W1 / CR4.12-P3 (reviewer-discovered ruff-RED-as-shipped). Finally closed when 4.12's reviewer measured the gate on a real POSIX host.
4. **Post-dispatch `check_stop` timing contract.** `check_stop` runs post-dispatch with a pre-dispatch state snapshot, forcing every trigger to re-read disk (`_ = state`). For 4.3/4.4 this made the natural integration-positive cell unreachable (forced monkeypatching `resolve_next_action`); for 4.7 it was a *safety* hole (a naive disk check fires after `rm -rf` already ran) → bridged to the 2B.6 pre-execution gate.
5. **Frozen `StopDecision` 4-field packing.** Every story's AC named a richer shape; all packed identifier into `target` + human summary into `reason`, never widening the dataclass.
6. **"Crash instead of halt gracefully" — exception-taxonomy misses.** Uncaught `WorkflowError` (4.10-D1), `SignoffError` escaping a too-narrow `except (OSError, ValueError)` (4.11), `UnicodeDecodeError ⊂ ValueError not OSError` (4.4, 4.8). Any trigger exception aborts the loop iteration → every trigger must fail-soft.

## 4. Systemic Discoveries (retro-owned, surfaced by the Epic 3 gates)

These are **not** repeats of Epic 3 mistakes — they are second-order findings the Epic 3 gates uncovered:

1. **`main` was never CI-green at the matrix tier (HIGH).** Applying the A2 branch-protection toggle exposed that all 5 workflows referenced a non-existent `astral-sh/setup-uv@v8` moving tag → every run failed at `Set up job` in ~10s and **no run ever reached the test step**. Every prior "green main / N tests pass / ≥87% coverage" claim was a single-interpreter local run. A 3.10-only `tomllib` import bug + a backlog of pre-existing failures rode through undetected. → Need a **"CI-never-started ≠ green" guard**.
2. **A1 guard false-passes on the Windows dev host (HIGH, cp1252).** `subprocess.run(..., text=True)` lacks `encoding="utf-8"`; UTF-8 em-dashes in commit subjects trigger a background `UnicodeDecodeError` under cp1252; `subprocess.run` returns rc=0 with empty stdout → the gate exits 0 validating nothing. Compounded by commit-msg hooks not installed in `.git/hooks/`. R1/R2 for every Epic 4 close-out had to be **verified by hand**. This reproduces the exact Epic 3 "looks-green-but-didn't-run" failure mode on a new platform. → One-line `encoding="utf-8"` fix + hook installation.

## 10. Key Takeaways

1. **Mechanized gates held perfectly** — zero commit-discipline regressions across 12 stories. The Epic 3 lesson generalized.
2. **A "green" signal is only as good as its execution path.** Both systemic discoveries (CI-never-green; cp1252 gate false-pass) are the same root: trusting a gate signal that never actually flowed. Generalize the Epic 3 lesson to *"green locally ≠ green in CI"* and *"a gate that can't run on the host it runs on is not a gate."*
3. **The defer backlog is layered, not flat** — most defers gate behind one root (mock-dispatch); isolating that let us ride it cleanly while closing the genuinely independent security + coverage items before Epic 5.
4. **Spec-prose ≠ on-disk reality.** The naming-drift trap class is now explicit; verify load-bearing symbols against source.
